# Dictionary Validation

Ensure consistent terminology using custom dictionaries.

## Overview

DocuSanity's dictionary validation feature helps you maintain consistent terminology across your documentation. It allows you to define custom dictionaries of approved terms and automatically checks your documentation for deviations.

## Benefits

- **Consistency**: Ensure consistent use of technical terms and product names.
- **Accuracy**: Reduce errors and improve the accuracy of your documentation.
- **Clarity**: Make your documentation easier to understand for all readers.
- **Professionalism**: Maintain a professional and polished appearance.

## Setting Up Custom Dictionaries

1.  Create a JSON file for each custom dictionary.
2.  Add an array of approved terms to each file.
3.  Configure the `dictionary` check in your `docusanity.config.js` file.

### Example Dictionary File

```json
// .docusanity/dictionaries/technical-terms.json
[
  "API",
  "CLI",
  "SDK",
  "YAML",
  "JavaScript",
  "Markdown"
]
```

### Configuration

```javascript
// docusanity.config.js
module.exports = {
  checks: {
    dictionary: {
      enabled: true,
      customDictionaries: [
        '.docusanity/dictionaries/technical-terms.json'
      ],
      caseSensitive: false,
      ignorePatterns: [
        '\`\`\`[\\s\\S]*?\`\`\`', // Ignore code blocks
        'https?://[^\s]+'       // Ignore URLs
      ]
    }
  }
};
```

## Running Dictionary Validation

You can run dictionary validation in several ways:

### Command Line

```bash
docusanity check dictionary
```

### Programmatically

```javascript
const { Validator } = require('@docusanity/validator');

const validator = new Validator('./docs');
validator.checkDictionary()
  .then(results => console.log(results))
  .catch(error => console.error(error));
```

### GitHub Actions

You can also integrate dictionary validation into your CI/CD pipeline using [GitHub Actions](/docs/github-actions).

## Common Issues

- **Case Sensitivity**: Ensure your configuration matches your needs
- **Exclusions**: Verify that your exclusions are working as expected
- **Dictionary Format**: Ensure your dictionary files are valid JSON

## Best Practices

- Regularly update your custom dictionaries
- Use a consistent naming convention for dictionary files
- Test your configuration thoroughly
- Integrate dictionary validation into your CI/CD pipeline

Visit [Custom Dictionaries](/docs/dictionary-validation/custom) for more detailed information.
